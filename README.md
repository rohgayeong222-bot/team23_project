# 로그 기반 서비스 모니터링 시스템 (Log-based Service Monitoring System)

본 프로젝트는 실제 운영 환경과 동일한 리눅스(Ubuntu 24.04 LTS) 환경에서 차세대 웹 서버인 Nginx의 에러 로그를 실시간으로 추적, 파싱, 분석하여 시스템 상태를 모니터링하고 가시화하는 실실시간 파이프라인 시스템입니다.

## 2. 시스템 아키텍처 및 핵심 기능 (Core Pipeline)
본 시스템은 역할 기반으로 모듈화된 다중 C 파일 구조를 가지며, 제안서의 5대 핵심 파이프라인 및 추가 복잡도 요건을 100% 충족하여 구현되었습니다.

1. **실시간 데이터 연동 (Real-time Integration)**: 가상 데이터(Dummy Logs) 환경을 넘어 로컬에 구축된 실제 Nginx 웹 서버의 리얼 타임 에러 로그(`.log`)를 직접 연동하여 분석합니다.
2. **점진적 로그 추적 (Incremental Tracking)**: 매번 파일 전체를 읽는 비효율을 제거하고, 파일의 크기 변화를 메타데이터 수준에서 감지하여 새롭게 추가된 로그만 최적화하여 점진적으로 파싱합니다.
3. **지표 계산 및 상태 판정 (Metric Calc & Analysis)**: 전체 요청 수 대비 에러 발생 빈도를 실시간 계산(Error Ratio)하여 시스템 상태를 3단계(`HEALTHY(0)`, `WARNING(1)`, `CRITICAL(2)`)로 동적 판정합니다.
4. **종합 요약 리포트 (Summary Report)**: 누적 분석 결과를 사람이 읽기 쉬운 요약 지표 리포트 형식(`summary.txt`)으로 기록합니다.
5. **사건 상시 대응 리포트 (Incident Review)**: 에러율이 임계치(30%)를 초과하여 `CRITICAL` 상태에 진입하는 즉시 이상 징후 구간의 최초 발생 시점과 에러 상세 원인을 기록하는 비상 리포트(`incident.txt`)를 자동 생성합니다.
6. **날짜별 데이터 파티셔닝 (Data Partitioning)**: 장기 누적 데이터 관리 효율성을 극대화하기 위해, 리포트 저장 시 연-월-일 포맷의 디렉토리를 동적으로 생성하여 아카이빙합니다.
7. **멀티스레드 네트워크 서버 (Bonus - Socket & pthread)**: 메인 추적 엔진과 별도로 POSIX 스레드를 분리하여 8080번 포트에서 동작하는 소켓 서버를 구동, 외부 클라이언트의 실시간 지표 요청에 유연하게 대응합니다.
