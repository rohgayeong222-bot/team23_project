# 로그 기반 서비스 모니터링 시스템 (Log-based Service Monitoring System)

본 프로젝트는 실제 운영 환경과 동일한 리눅스(Ubuntu 24.04 LTS) 환경에서 차세대 웹 서버인 Nginx의 에러 로그를 실시간으로 추적, 파싱, 분석하여 시스템 상태를 모니터링하는 멀티스레드 기반 인프라 파이프라인 시스템입니다.

## 2. 시스템 아키텍처 및 핵심 고도화 요소
본 시스템은 가영 PM의 주도하에 설계되었으며, 실무급 인프라 요건과 데이터 보존 제약을 완벽하게 해결하도록 리팩토링되었습니다.

1. **비동기 멀티스레딩 (POSIX pthread)**:
   * **Thread 1 (메인 작업반장)**: Nginx 에러 로그(`error.log`) 파일의 변경 사항을 메타데이터 수준(`stat`)에서 무한 루프로 감시하고 분석 및 리포팅을 담당합니다.
   * **Thread 2 (안내데스크 소켓 서버)**: 메인 흐름과 완벽히 독립되어 **8080번 포트 소켓**을 열고 대기하며, 외부 대시보드나 클라이언트의 실시간 지표 요청에 비동기적으로 즉시 응답합니다.
2. **리얼 Nginx 인프라 연동**: 가상의 더미 데이터를 탈피하고 로컬 우분투에 세팅된 실제 Nginx 웹 서버의 리얼타임 로그 스트림을 파이프라인에 직결했습니다.
3. **날짜별 데이터 파티셔닝**: 누적 데이터 유실 및 관리 복잡도를 개선하기 위해 연-월-일(`YYYY-MM-DD`) 형식의 하위 디렉토리를 실시간으로 동적 생성하여 아카이빙 구조를 고도화했습니다.
4. **30일 데이터 보존 정책 (Retention Policy / Log Rotation)**: 서버 디스크 용량 고갈 방지를 위해 시스템 시작 시 30일이 지난 오래된 리포트 디렉토리를 탐색하여 자동으로 영구 정제하는 매니지먼트 아키텍처를 도입했습니다.
