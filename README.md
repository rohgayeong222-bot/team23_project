# Nginx 웹 서버 로그 모니터링 시스템

본 프로젝트는 리눅스(Ubuntu 24.04 LTS) 환경에서 구동되는 Nginx 웹 서버의 에러 로그를 실시간으로 추적하고 분석하여 시스템 상태를 모니터링하는 멀티스레드 기반 프로그램입니다.

## 2. 주요 기능 및 시스템 구조
본 프로젝트는 시스템 프로그래밍에서 배우는 핵심 개념(멀티스레드, 소켓 통신, 저수준 파일 입출력 등)을 활용하여 구현되었습니다.

1. **멀티스레드 기반 비동기 처리 (pthread)**:
   * **로그 감시 스레드 (메인)**: Nginx 에러 로그(`error.log`) 파일의 변경 사항을 실시간으로 감시하고, 에러율을 계산하여 리포트를 작성합니다.
   * **소켓 통신 스레드 (서브)**: 메인 스레드와 독립적으로 8080번 포트에서 네트워크 통신을 대기하며, 웹 브라우저나 외부 클라이언트의 상태 확인 요청에 실시간 통계 데이터를 JSON 형태로 응답합니다.
2. **실제 Nginx 서버 연동**: 가상의 더미 데이터가 아닌, 로컬 환경에 설치된 실제 Nginx 웹 서버의 로그 파일(`/var/log/nginx/error.log`)을 직접 읽어와 분석합니다.
3. **날짜별 리포트 분리 저장**: 데이터 관리를 쉽게 하기 위해 연-월-일(`YYYY-MM-DD`) 형식으로 폴더를 매일 자동으로 생성하여 요약(summary) 및 비상(incident) 리포트를 나누어 저장합니다.
4. **오래된 로그 자동 삭제 (보존 정책)**: 서버의 하드디스크 용량이 가득 차는 것을 방지하기 위해, 프로그램 시작 시 30일이 지난 오래된 날짜의 리포트 폴더는 시스템 명령어를 통해 자동으로 삭제되도록 구현했습니다.
